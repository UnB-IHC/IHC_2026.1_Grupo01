# 🛠️ Desenvolvimento Web

Análise técnica dos critérios de acessibilidade relacionados ao código, HTML, navegação e funcionalidade. Este guia estabelece os parâmetros técnicos necessários para garantir a conformidade do projeto com as normas de acessibilidade vigentes.

---

## 📋 Como documentamos aqui (instruções internas)

> 🗑️ **[REMOVER NA VERSÃO FINAL - SEÇÃO INTEIRA]** Delete quando terminar.

### O que vocês devem fazer nessa página

Aqui você deve documentar **critérios técnicos focados em desenvolvimento e código**. Isso inclui:

- Semântica HTML e estrutura
- Navegação com teclado e ordem de tabulação
- Testes com leitores de tela (ARIA quando necessário)
- Validação de erros e mensagens
- Performance e funcionalidade

Os items [EXEMPLO] no checklist **devem ser trocados** por critérios reais que você vai validar no seu projeto.

### Como customizar

Remova cada `[EXEMPLO]` e substitua por um critério real do **WCAG 2.2, NBR 17225 ou Guia UK-BR** que seja relevante para o desenvolvimento do seu projeto:

❌ Errado: `[EXEMPLO] Indicador de foco visível em botões` (isto é apenas exemplo)

✅ Certo (exemplo de como fica):

- `Semântica HTML correta com ARIA quando necessário`
- `Teclado: todos os controles interativos acessíveis`
- `Ordem de tabulação lógica e intuitiva`

**Lembrete**: Os nomes dos critérios devem vir exatamente de WCAG 2.2, NBR 17225 ou Guia UK-BR, não inventados.

# 🔧 Ferramentas para Testar Acessibilidade

Esta página reúne **ferramentas práticas** para apoiar o time (Desenvolvimento, Design, Conteúdo, QA) na verificação de acessibilidade.

> **Importante:** Validações automáticas **não substituem** testes manuais com **teclado** e **tecnologias assistivas** (leitores de tela), nem **testes com pessoas reais**. Use as ferramentas como complemento, não como solução completa.

## 7. Testes Manuais de Teclado (Sem Ferramenta)

**Quando usar:** **Sempre** — é a **base** de operabilidade.

**Como fazer (roteiro prático):**

1. **TAB do topo ao rodapé** (sem mouse)
2. **Acesse todos os interativos** — botões, links, inputs, dropdowns, modais, tabs
3. **Verifique:**
   - Foco é sempre **visível e clara**?
   - Ordem de TAB é **lógica** (esquerda→direita, topo→base)?
   - Não há **prisão de foco** em modais ou menus?
   - **Esc** fecha modais?
   - **Shift+Tab** volta corretamente?

**Limitações:** Manual e requer atenção. Pode falhar em detalhes.

**Dica avançada:** Grave um **GIF curto** percorrendo os elementos com foco visível — anexar ao PR facilita o code review.
